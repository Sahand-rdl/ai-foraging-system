## Actors

* **User:** AI practitioner or researcher, the primary user of the tool. A technical user who explores, designs, and develops AI/ML solutions.
* **New User:** Another user, who accesses the Repository in order to reuse Artifact Knowledge.

## Core concepts

| **Concept** | **Definition** | **Examples** |
| ------ | ----- | ------- |
| ML Problem | The definition of the ML problem that a User is having | Neural Network |
| Search Tool | A tool that the user uses to search for Knowledge Source and Knowledge Artifact | Google, Google Scholar |
| Knowledge Source | The origin of information which contains the Knowledge Artifact | Blog entry, academic publication, documentation, git repo |
| Knowledge Artifact | A specific piece of knowledge from the Knowledge Source that can be evaluated and reused | Code snippet, pretrained model, description of an algorithm |
| Source Metadata | Data describing a Knowledge Source | Author, publisher, publication date, number of citations |
| Evaluation Approach | A machine (?)  | A preexisting model that can classify (?) |
| Evaluation Result | The outcome of the machine's and the user's assessment of a Knowledge Artifact or Knowledge Source, based on an approach | Credibility, reusability, relevance |
| Repository | A place(?) where the relevant Knowledge Source and Knowledge Artifcact is stored, where it is accessible by the new user. | Credibility, reusability, relevance |

## Activities

1. The User outlines his current ML Problem.
2. The User uses the Search Tool to search Knowledge Artifact.
3. The Search Tool shows various Knowledge Source.
4. The Knowledge Source contains (implicitly) Source Metadata and includes Knowledge Artifact
5. Evaluation Approach labels the Knowledge Source and Knowledge Artifact
6. The User evaluates the Knowledge Source and Knowledge Artifact (give x/5 Star, and optionally a comment) (still to be determined!)
7. The relevant Knowledge Source, Knowledge Artifact and Evaluation Result are stored in the Repository
8. A New User can easily access this based on their ML Problem
